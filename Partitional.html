OVERVIEW

The goal of clustering is to find a structure in a collection of unlabelled data. In this case, it is used to separate the noise from the CH model by forming semantically meaningful groupings that contain points with similarities called clusters. Ideally, these clusters will correspond to a collection of points that make sense to humans, such as all the points that comprise the scaffolding in the three-dimensional church model. Therefore, the taxing task of point cloud cleaning can be significantly simplified if the aforementioned is achieved. The clustering techniques explored are partitional algorithms and a Gaussian mixture model. K-means is the baseline clustering algorithm in which the three aforementioned algorithms can be compared.

Partitional algorithms assign a centroid to each group and then iteratively allocate the closest points to that centroid to form a cluster.
A Gaussian mixture model is a soft clustering algorithm that assigns various probabilities to points, making the algorithm more flexible. Every component (Gaussian density) has its own mean, as well as covariance. Clusters are formed due to different Gaussian distributions.

Research Objectives

  1. Does K-medians Gaussian mixture model and Fuzzy c-means produce clusters on point clouds of CH sites that are more semantically meaningful than those produced by k-means algorithm, using cluster validity metrics of Silhouette Coefficient and DB Index?
  2. What is the upper bound accuracy when using the resulting clusters to perform binary classification using' keep' and' discard' labels on a point cloud?
  3. Can clustering simplify and accelerate the mainly-manual point cloud cleaning task?

IMPLEMENTATION

The design of our approach is split into a five step pipeline:

  1. The dataset used throughout the project is 3D point clouds from TLS campaigns conducted by the VCG at CNR-ISTI. The recordings from the TLS scans are used as input in our pre-processing pipeline.
  2. We implement feature extraction on the raw point cloud to form two rich datasets.

    • Raw dataset: (x,y,z) , intensity
    • CloudCompare dataset: (x,y,z), intensity, 18 human interpretable features
    • PointNet++ dataset: (x,y,z), 128 abstract features
  3. Clustering is then perfomed on the 3 datasets to decompose the 3D point cloud into different segments.
  4. Visual inspection is implemented using a point cloud processing tool kit.
  5. We then analyse and evaluate the prodcued clusters numerically through Cluster validity metrics (silhouette score, BIC criterion and DB index) and classification performance metrics (F1 score, recall, precision and Intersection over Union metric).
